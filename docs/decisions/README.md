# Decision Records

Short records of architectural and product decisions. Each one captures the context at the time, what was decided, and why. They are not updated after the fact — if a decision changes, add a new record that supersedes the old one.

Format: tight. Context in 2–4 sentences, decision in one, rationale as bullets, consequences as bullets. If it needs more than a page, it probably needs to be split.
